Sass

Overview

Sass is a CSS pre-processor, meaning that it extends the CSS language, adding features that allow variables, mixins, functions and many other techniques that allow you to make CSS that more maintainable, themeable and extendable.

To learn more about Sass, please visit: http://sass-lang.com/

Structure

To allow simpler editing we’ve divided the spreadsheet into smaller files:

  • _blog.scss (styles for blog)
  • _buttons.scss (styles for buttons)
  • _forms.scss (styles for forms)
  • _portfolio.scss (styles for portfolio)
  • _preloader.scss (preloader styles)
  • style.scss (main style sheet file)
  • _variables.scss (contains Sass variables like colors etc.)
  • zupa.scss (base and shared styles)

If you don’t need one of the modules, you simply remove it from the style.scss file and recompile the spreadsheet.

CSS

If you don’t feel comfortable with Sass files, you can still adjust the template to your needs in traditional way by editing css/style.css file.

Animations

animate.css

Animations are accessible through animate.css file. It is possible to start animations whilst scrolling. For this purpose all you have to do is to add data-acos attribute to html element, like so:

Example
<p data-acos="fadeInUp" data-acos-delay=".5s">Lorem ipsum</p>

Available options:

  • data-acos (required) – class name of animation
  • data-acos-delay (optional) – delay of animation
  • data-acos-duration (optional) – duration of animation
  • data-acos-repeat (optional) – choose whether animation should fire once, or every time you scroll up/down to element

If you require more advanced options, consider using wow.js

Animated inputs

We’ve prepared a style that allows animation during activation/deactivation of chosen fields of input[text] and textarea type. In order to use this style on your chosen element, encircle it with a container and add .animated-form-control class.

Example
<div class="animated-form-control">
	<input placeholder="Login" type="text" name="name" class="text form-control" required="required" aria-required="true" />
</div>

Maps

To use the Google Maps, you must register your project on the Google API Console and get a Google API key which you can add to your website.

To learn how get your Google API key, please visit: https://developers.google.com/maps/documentation/javascript/get-api-key

When loading the Google Maps JavaScript API, substitute YOUR_API_KEY in the code below with the API key you got from the previous step.

Code
<script src="//maps.google.com/maps/api/js?sensor=false&key=YOUR_API_KEY"></script>

You can find this code in contact.html file.
